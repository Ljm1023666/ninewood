改造范围

冻结（不动）

  • Discover — 首页公开入口，ScrollNavbar + Hero 已成熟
  • Login — 认证入口，动效与品牌表达已到位

改造（Layout 内 33 个视图）

设置、订单、消息、圈子、搜索认证、卡池、个人页、运营工具等——通过 9 类页面模板 覆盖，而非逐页重设计。

气质：Achromatic Precision × Ninewood 基因

v2 问题与产品脱节

  • C 类做成双栏文章详情，丢失居中 3D 翻面需求卡
  • G 类做成个人主页资产网格，丢失卡包/手牌/桌面三层结构
  • 未引用 DemandCardInner、BlackPackSpine 等真实组件

v3 原则从现有实现生长

  • 硬极简：OLED 纯黑 + 发丝线，无 SaaS 阴影徽章
  • 保留辨识度：卡池隐喻、需求翻面卡、ListItemCard 列表不删
  • 统一壳层：内部页收敛顶栏/分组/mono 标签，不动发现/登录
  • accent 克制：#3388FF 每屏一处主焦点

如何用 Stitch 定方向（当前阶段）

1. 本地总览

打开 docs/ui-ux-audit-preview.html，先看 产品基因 表，再对照九类 v3 稿是否像 Ninewood。

2. 高保真交互

点击各卡片下的 HTML 预览，在浏览器里感受布局、间距、层次（需联网）。

3. Stitch 里迭代

登录 stitch.withgoogle.com → 项目 Ninewood UI/UX Audit → 选中 Screen → 改 prompt 或 Generate Variants。

4. 你确认后我再动

重点评审 v3 的 B/C/F/G 是否像 Ninewood；确认后我再推进其余类 v3 或进入 React，当前仍不写代码。
